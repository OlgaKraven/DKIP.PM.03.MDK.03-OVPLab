Лабораторная работа №5

Тема: Адаптивный дизайн и UX-аудит

Теория Задание Контрольные вопросы

1. Теоретическая часть

1.1. Что такое адаптивный дизайн

Адаптивный дизайн — это подход к созданию веб-страниц, при котором содержимое, макет и функции автоматически подстраиваются под размер экрана устройства (десктоп, планшет, мобильный телефон).

1.2. Медиа-запросы (Media Queries)

Механизм CSS, позволяющий применять стили в зависимости от характеристик устройства (ширина экрана, ориентация, разрешение).

@media (max-width: 768px) {
  .container { width: 100%; }
  nav ul { flex-direction: column; }
}

1.3. Что такое UX (User Experience)

UX — это общий опыт пользователя при взаимодействии с продуктом или сервисом. В контексте веб-сайтов — это удобство навигации, скорость загрузки, понятность интерфейса, доступность.

1.4. Основные принципы UX

  • Простота и интуитивность — пользователь должен понимать, как что-то сделать.
  • Доступность — сайт должен быть удобен для всех, включая людей с ограниченными возможностями.
  • Скорость — быстрая загрузка страниц.
  • Ясность — чёткие заголовки, кнопки, инструкции.

2. Задание

2.1. Подготовка

Возьмите сайт из предыдущей лабораторной (например, my-site из ЛР4).

2.2. Добавление адаптивности

  1. Определите 3 брейкпоинта: ≥1024px (десктоп), 768px–1023px (планшет), <768px (мобильный).
  2. Используйте Flexbox или Grid для создания адаптивных сеток.
  3. Примените медиа-запросы для изменения размеров шрифтов, отступов, навигации.

2.3. UX-аудит

Протестируйте сайт на устройствах или в DevTools (Device Toolbar). Проверьте:

  • удобство навигации (размеры кнопок/ссылок),
  • читаемость текста,
  • корректность отображения изображений,
  • элементы, снижающие конверсию (лишние шаги, скрытые ошибки форм).

2.4. «Скорость = деньги»

Замерьте время загрузки главной страницы (DevTools → Network). Объясните, как задержка снижает продажи (пример: уход после 3 секунд ожидания).

2.5. «Битва конкурентов»

Сравните демо-сайт ReadyScript с крупным магазином (Ozon/Wildberries) по 3 параметрам:

  • скорость загрузки,
  • удобство поиска,
  • визуальная привлекательность.

3. Контрольные вопросы

  1. Какие события браузера удобно использовать для детекции активности/бездействия?
  2. Как организовать адаптивную вёрстку для трёх брейкпоинтов (≥1024, 768–1024, <768)?
  3. Какие стили обычно меняют при ночной теме и как её переключать программно?
  4. Как избежать «дребезга» (частых вызовов) при обработке событий мыши/тача?
  5. Чем полезен онлайновый журнал событий и как безопасно добавлять строки в DOM?
Подсказка: связывайте находки DevTools/UX-аудита с бизнес-метриками (конверсия, удержание, выручка, CAC, LTV) и приоритизируйте улучшения.
