Итоговый проект: аудит и презентация

Теория Задания Контрольные вопросы

1. Теоретическая часть

Аудит веб-приложения — это комплексная проверка качества сайта по техническим, визуальным и маркетинговым критериям.

Основные направления аудита:

  • Техническая оптимизация — скорость загрузки, кэширование, адаптивность, структура кода.
  • SEO-аудит — теги title, description, карта сайта, robots.txt, корректность ссылок.
  • Контент-аудит — качество текстов, структура, уникальность, читаемость.
  • Графический аудит — вес изображений, форматы, адаптация под Retina.
  • Аналитический аудит — корректность UTM-меток, подключение аналитики, работа фильтров и отчётов.

2. Практические задания

2.1. Выбор проекта для аудита

  1. Выберите любой ранее выполненный проект (например, сайт из ЛР5–ЛР8).
  2. Убедитесь, что он доступен онлайн (CodePen / GitHub Pages / Replit).
  3. Скопируйте ссылку для анализа.

2.2. Технический аудит

  1. Проверьте сайт в PageSpeed Insights или GTmetrix.
  2. Зафиксируйте показатели: Performance, Accessibility, Best Practices, SEO.
  3. Составьте таблицу:
Показатель Значение Рекомендация по улучшению
Performance
Accessibility
SEO

2.3. Контент и визуальный аудит

  1. Проверьте наличие:
    • заголовков h1–h3, списков, структурированных текстов;
    • корректных alt-описаний у изображений;
    • единых шрифтов и отступов;
    • адаптивности при ширине 360–1920 px.
  2. Сделайте скриншоты проблемных мест до и после исправления.

2.4. SEO-аудит

  1. Проверьте:
    • корректность тегов <title>, <meta description>;
    • наличие и правильность sitemap.xml и robots.txt;
    • структуру URL.
  2. Оцените соответствие требованиям поисковой оптимизации.

2.5. Анализ рекламных меток

  1. Проверьте корректность UTM-меток в ссылках.
  2. Составьте таблицу:
Ссылка Источник Кампания Проверка
/promo?utm_source=vk&utm_campaign=salevksale✅ работает
/blog?utm_source=telegram&utm_medium=socialtelegramsocial⚠ отсутствует utm_term

2.6. Подготовка презентации

Создайте презентацию из 5–7 слайдов (Google Slides / Canva / PowerPoint Online):

  • Тема и цель проекта.
  • Скриншоты сайта.
  • Основные результаты аудита (до / после).
  • Методы оптимизации, применённые на сайте.
  • Выводы и рекомендации.

2.7. Вывод и оформление отчёта

В отчёте отразите:

  • цели аудита,
  • инструменты проверки,
  • таблицы с результатами,
  • скриншоты,
  • итоговый анализ и рекомендации по улучшению.
